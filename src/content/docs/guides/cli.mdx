`nokkvi <verb>` sends a single command to the running Nokkvi instance over a Unix socket and exits. Use it from shell scripts, window manager keybinds, or any external tool ‚Äî there's no GUI startup cost because the CLI path returns before iced initializes.

The CLI is an alternative to [MPRIS](/guides/mpris/). MPRIS covers the standard transport surface and works with any MPRIS controller; the CLI adds verbs that MPRIS doesn't expose (`switch-view`, `love`, `rate`, `clear-queue`, `consume`).

## Verbs

### Transport

| Verb | Arg | Action |
| :--- | :--- | :--- |
| `play` | ‚Äî | Start playback |
| `pause` | ‚Äî | Pause playback |
| `play-pause` | ‚Äî | Toggle play / pause |
| `stop` | ‚Äî | Stop and reset the playhead |
| `next` | ‚Äî | Skip to the next track |
| `previous` | ‚Äî | Return to the previous track |
| `seek` | `<seconds>` | Seek to absolute position (float, in seconds) |
| `volume` | `<value>` | Set volume ‚Äî see [Volume syntax](#volume-syntax) |

### Queue and modes

| Verb | Arg | Action |
| :--- | :--- | :--- |
| `shuffle` | ‚Äî | Toggle random mode |
| `repeat` | ‚Äî | Cycle repeat: off ‚Üí one ‚Üí queue |
| `consume` | ‚Äî | Toggle consume mode |
| `clear-queue` | ‚Äî | Empty the queue and stop playback |
| `add-to-queue` | ‚Äî | Add the focused list item to the queue ‚Äî the in-app Shift+A |
| `remove-from-queue` | ‚Äî | Remove the centered queue song ‚Äî Ctrl+D; errors `not_in_queue_view` outside the queue view |

### View

| Verb | Arg | Action |
| :--- | :--- | :--- |
| `switch-view` | `<view>` | Switch the top pane ‚Äî see [View names](#view-names) |

### Navigation

| Verb | Arg | Action |
| :--- | :--- | :--- |
| `nav-up` | ‚Äî | Move the focused list selection up ‚Äî Backspace |
| `nav-down` | ‚Äî | Move the focused list selection down ‚Äî Tab |
| `enter` | ‚Äî | Activate the centered item ‚Äî play / expand / edit |

`nav-up`, `nav-down`, and `enter` are fire-and-forget; the move is async, so query `selection` afterward to read where focus landed.

### Query

These are pure reads ‚Äî they print JSON and change nothing.

| Verb | Arg | Action |
| :--- | :--- | :--- |
| `status` | ‚Äî | Print playback state, track, album, position, duration, volume, and the three modes as JSON |
| `selection` | ‚Äî | Print the focused view's centered item as JSON: view, kind, name, artist, rating, starred; kind is null when nothing is selectable |

### Playing track

These act on whatever's currently playing and exit `1` with a `no_playing_track` error if nothing is loaded.

| Verb | Arg | Action |
| :--- | :--- | :--- |
| `love` | ‚Äî | Toggle Navidrome star on the playing track |
| `rate` | `<delta-or-abs>` | Adjust the playing track rating ‚Äî see [Rating syntax](#rating-syntax) |

### Misc

| Verb | Arg | Action |
| :--- | :--- | :--- |
| `ping` | ‚Äî | Probe the running instance. Prints `pong` and exits `0`. |
| `-h`, `--help` | ‚Äî | Print usage and the full verb catalog, then exit `0`. |
| `-V`, `--version` | ‚Äî | Print `nokkvi <version>`, then exit `0`. |

## Argument syntax

### Volume syntax

The `<value>` arg accepts two shapes:

- **Delta** ‚Äî `+N` or `-N`, added to the current volume and clamped silently to `0.0..=1.0`. Repeated `volume +0.05` at the ceiling is a no-op.
- **Absolute** ‚Äî `N` in `0.0..=1.0`. Out-of-range values error rather than clamp.

```bash
nokkvi volume +0.05    # bump 5%
nokkvi volume -0.1     # drop 10%
nokkvi volume 0.5      # set to 50%
```

### Rating syntax

The `<delta-or-abs>` arg accepts two shapes:

- **Delta** ‚Äî `+N` or `-N`, added to the current rating and clamped to `0..=5`.
- **Absolute** ‚Äî `0` through `5`. Out-of-range values error.

```bash
nokkvi rate +1    # bump by one star
nokkvi rate -1    # drop by one star
nokkvi rate 5     # set to five stars
nokkvi rate 0     # clear the rating
```

To confirm each change with a desktop notification ‚Äî useful when you've bound these to a window-manager keybind ‚Äî enable [`rating_change_notification_enabled`](/reference/config/#rating-reminder).

### View names

`switch-view` accepts one of: `albums`, `queue`, `songs`, `artists`, `genres`, `playlists`, `radios`, `harbour`, `settings`. Anything else errors with the full list in the message.

```bash
nokkvi switch-view albums
nokkvi switch-view queue
nokkvi switch-view harbour   # jump to the Harbour home view
```

## How it connects

Each running Nokkvi daemon binds a per-PID socket so multiple instances can coexist:

| Environment | Socket path |
| :--- | :--- |
| `$XDG_RUNTIME_DIR` set | `$XDG_RUNTIME_DIR/nokkvi-{pid}.sock` |
| Unset (fallback) | `/tmp/nokkvi-{uid}-{pid}.sock` |

The `{uid}` in the fallback path comes from the `$UID` environment variable, not the kernel uid. When `$UID` is unset it falls back to the literal `default` (e.g. `/tmp/nokkvi-default-{pid}.sock`).

The CLI auto-discovers the live socket ‚Äî you don't pass a path. If no instance is running, the command exits `1` with a message listing the directory it searched.

## Exit codes

| Code | Meaning |
| :--- | :--- |
| `0` | Server accepted the verb. Any payload is printed to stdout (e.g. `ping` prints `pong`). |
| `1` | No live instance found, or the server returned an error (e.g. `invalid_args`, `no_playing_track`, `unavailable`, `not_in_queue_view`). Error message goes to stderr. |

## Hyprland keybinds

Hyprland's [Lua config](https://wiki.hypr.land/Configuring/Lua-API/) is the current binding API ‚Äî `hl.bind(keys, hl.dsp.exec_cmd(cmd), opts?)` matches the shape from the upstream `example/hyprland.lua`. The Lua block below is the author's actual `~/.config/hypr/keybinds.lua` for nokkvi; the legacy `.conf` tab is for stable distros still on the older Hyprland.

<Tabs syncKey="hypr-config">
  <TabItem label="Lua (current)">
    ```lua
    -- ~/.config/hypr/keybinds.lua
    local mod_alt = "SUPER + ALT"

    -- Transport
    hl.bind(mod_alt .. " + space", hl.dsp.exec_cmd("nokkvi play-pause"))
    hl.bind(mod_alt .. " + right", hl.dsp.exec_cmd("nokkvi next"))
    hl.bind(mod_alt .. " + left",  hl.dsp.exec_cmd("nokkvi previous"))

    -- Volume ‚Äî { repeating = true } repeats while held
    hl.bind(mod_alt .. " + up",    hl.dsp.exec_cmd("nokkvi volume +0.01"), { repeating = true })
    hl.bind(mod_alt .. " + down",  hl.dsp.exec_cmd("nokkvi volume -0.01"), { repeating = true })

    -- Rating + love on the currently playing track
    hl.bind(mod_alt .. " + L",     hl.dsp.exec_cmd("nokkvi love"))
    hl.bind(mod_alt .. " + equal", hl.dsp.exec_cmd("nokkvi rate +1"))
    hl.bind(mod_alt .. " + minus", hl.dsp.exec_cmd("nokkvi rate -1"))
    ```
  </TabItem>
  <TabItem label="Legacy .conf">
    ```conf
    # ~/.config/hypr/hyprland.conf

    # Transport
    bind  = $mainMod ALT, space, exec, nokkvi play-pause
    bind  = $mainMod ALT, right, exec, nokkvi next
    bind  = $mainMod ALT, left,  exec, nokkvi previous

    # Volume (binde repeats while held)
    binde = $mainMod ALT, up,   exec, nokkvi volume +0.01
    binde = $mainMod ALT, down, exec, nokkvi volume -0.01

    # Rating + love on the currently playing track
    bind  = $mainMod ALT, L,     exec, nokkvi love
    bind  = $mainMod ALT, equal, exec, nokkvi rate +1
    bind  = $mainMod ALT, minus, exec, nokkvi rate -1
    ```
  </TabItem>

Add `{ locked = true }` (Lua) or use `bindl`/`bindle` (`.conf`) on any bind you want to keep working while the session is locked ‚Äî the upstream example sets it on `XF86Audio*` keys.

## Other window managers

The CLI is a plain executable ‚Äî use whatever your compositor's exec-keybind syntax is (`sway`'s `bindsym`, `i3`'s `bindsym`, `dwm`'s `config.h`, etc.) and call `nokkvi <verb>` from it.
