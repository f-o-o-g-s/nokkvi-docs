---
title: Media Controls (MPRIS)
description: Control Nokkvi from your desktop, scripts, or window manager using MPRIS and playerctl.
---

Nokkvi registers as `nokkvi` on the MPRIS D-Bus interface. Any MPRIS-compatible controller — `playerctl`, your desktop's media keys, or a window manager keybind — will work.

For verbs MPRIS doesn't expose (`switch-view`, `love`, `rate`, `clear-queue`, `consume`), see the [`nokkvi` CLI](/guides/cli/).

## CLI

```bash
playerctl -p nokkvi play-pause
playerctl -p nokkvi next
playerctl -p nokkvi previous
playerctl -p nokkvi volume 0.05+
playerctl -p nokkvi metadata
```

## Hyprland keybinds

Hyprland's [Lua config](https://wiki.hypr.land/Configuring/Lua-API/) is the current binding API. `hl.bind(keys, hl.dsp.exec_cmd(cmd), opts?)` is the canonical shape — the same one the upstream `example/hyprland.lua` uses for multimedia keys.

```lua
-- ~/.config/hypr/keybinds.lua
local mod_alt = "SUPER + ALT"

hl.bind(mod_alt .. " + space", hl.dsp.exec_cmd("playerctl -p nokkvi play-pause"))
hl.bind(mod_alt .. " + right", hl.dsp.exec_cmd("playerctl -p nokkvi next"))
hl.bind(mod_alt .. " + left",  hl.dsp.exec_cmd("playerctl -p nokkvi previous"))
hl.bind(mod_alt .. " + up",    hl.dsp.exec_cmd("playerctl -p nokkvi volume 0.01+"), { repeating = true })
hl.bind(mod_alt .. " + down",  hl.dsp.exec_cmd("playerctl -p nokkvi volume 0.01-"), { repeating = true })
```

`{ repeating = true }` replaces the old `binde` (`e` for "repeating") from the legacy `.conf` syntax. Drop the option for one-shot binds.

## Other window managers

The `playerctl` commands above work the same in any WM. Bind them however your compositor handles exec keybinds (e.g. `sway`'s `bindsym`, `i3`'s `bindsym`, `dwm`'s config.h).
