The Radio view manages internet radio stations on your Navidrome server. Stations are HTTP streams (Icecast/Shoutcast) that play directly ŌĆö not through the queue.

For navigation, search, and multi-select, see [Library Basics](/guides/library-basics/). Radio has no expansion, no ratings, and no Add-to-Queue / Add-to-Playlist actions.

## Sort and search

Stations only sort by **Name** (`Page Up` toggles direction). Search matches both station name and stream URL.

## Adding, editing, and removing stations

All radio CRUD lives in Nokkvi ŌĆö you don't need to touch Navidrome's web UI.

- **Add Station** button (header, top-right) opens a dialog for **name** and **stream URL**.
- **Right-click ŌåÆ Edit Station** re-opens that dialog pre-populated.
- **Right-click ŌåÆ Delete Station** removes the station after confirmation.
- **Right-click ŌåÆ Copy Stream URL** copies the URL to your clipboard.

Behind the scenes, Nokkvi calls Navidrome's `createInternetRadioStation`, `updateInternetRadioStation`, and `deleteInternetRadioStation` endpoints.

## Now playing

When a station is streaming, the metadata strip shows:

- The station name (with a radio-tower icon)
- The current track's **artist** and **title**, parsed from the stream's ICY metadata in real time
- The stream URL as a fallback if no ICY metadata has been received yet

Most public Icecast/Shoutcast streams advertise ICY metadata; if a station you've added shows only the URL with no track info, it likely doesn't.

While a radio is playing, `next` / `prev` (and the player-bar arrows) cycle through your stations in order, wrapping around at the ends ŌĆö they don't act on the queue.

## Scrobbling

Radio plays can scrobble **directly** to [ListenBrainz](https://listenbrainz.org/) and/or [Last.fm](https://www.last.fm/) ŌĆö straight from Nokkvi, not through Navidrome. Library scrobbling rides Navidrome's Subsonic relay (see [Navidrome ŌåÆ Scrobbling](/guides/navidrome/#scrobbling)), but a radio track has no library id for the server to forward, so Nokkvi submits it itself using credentials you connect on the client.

This is **off by default**. Turn it on under **Settings ŌåÆ Playback ŌåÆ Radio Scrobbling**.

### What gets scrobbled

Nokkvi reads the `Artist - Title` from the stream's [ICY metadata](#now-playing) ŌĆö the same text the metadata strip shows. Once a track has *actually played* for your listen threshold, it's submitted once; while it plays, an optional now-playing update is sent too.

- **Threshold** ŌĆö `Radio Listen Threshold` (config: [`radio_scrobble_threshold_secs`](/reference/config/#radio-scrobbling), default 60 s, range 20ŌĆō240). Radio streams don't report a duration, so this is a flat seconds count, not the percentage that [library tracks](/guides/navidrome/#scrobbling) use. Paused time doesn't count.
- **Now-playing** ŌĆö `Send Radio Now-Playing` (config: [`radio_now_playing_enabled`](/reference/config/#radio-scrobbling), on by default) pushes a live "now playing" to each connected service whenever the track changes, refreshed every 30 seconds.

<Aside type="note">
  A stream whose ICY title isn't in `Artist - Title` form (just a title, or nothing) can't be scrobbled ŌĆö Nokkvi needs both an artist and a title. A repeating track won't scrobble twice in a row; a new listen only registers when the stream title actually changes.
</Aside>

### Connecting a service

Credentials live in **Settings ŌåÆ Playback ŌåÆ Radio Scrobbling**, and each service's row shows whether it's connected. Set up either or both ŌĆö scrobbling fires for whichever you've connected.

- **ListenBrainz** ŌĆö open `ListenBrainz Token`, then paste a user token from your [ListenBrainz profile](https://listenbrainz.org/profile/). `Verify ListenBrainz` checks it works; clearing the field disconnects.
- **Last.fm** ŌĆö open `Last.fm API Credentials` and paste an API key + secret from [last.fm/api](https://www.last.fm/api/account/create), then run `Connect Last.fm` to authorize Nokkvi in your browser. `Disconnect Last.fm` clears the stored session.

These credentials are separate from any Last.fm/ListenBrainz keys configured on Navidrome for library scrobbling ŌĆö radio uses your own client-side credentials, not the server's.

<Aside type="note" title="Where credentials are stored">
  Radio credentials are saved to a plaintext `[radio_scrobble]` block in `config.toml`. You can edit it by hand, or set the `NOKKVI_RADIO_LISTENBRAINZ_TOKEN`, `NOKKVI_RADIO_LASTFM_API_KEY`, and `NOKKVI_RADIO_LASTFM_API_SECRET` environment variables to override the file ŌĆö handy for keeping secrets out of the config.
</Aside>

## Limitations

<Aside type="note">
  Radio playback is intentionally separate from the rest of Nokkvi.
</Aside>

- **No queue integration** ŌĆö radio streams aren't queued. Merely navigating to the Queue view leaves the radio playing; playing anything from the queue (or any library view) stops the stream and switches back to queue playback.
- **No server-side play counts** ŌĆö radio plays don't go through Navidrome, so they never bump play counts on your server. They *can* scrobble straight to ListenBrainz/Last.fm instead ŌĆö see [Scrobbling](#scrobbling) above.
- **No favorites or playlists** ŌĆö you can't star a station or add it to a playlist.
- **No artwork or expansion** ŌĆö radio rows are flat; no cover art, no track listings.
- **Session-only playback** ŌĆö the stream stops when you close Nokkvi and doesn't auto-resume.
