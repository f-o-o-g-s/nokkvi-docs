Nokkvi stores your preferences in `~/.config/nokkvi/config.toml`. While most of these can be toggled in the in-app **Settings** menu (Default: `` ` `` backtick), this guide provides a complete technical reference for each key.

:::tip[Verbose Config]
By default, Nokkvi only writes settings that you have modified to the file. You can toggle **Verbose Config** in the Settings menu to force Nokkvi to write *every* default value into the file for easier manual editing.
:::

:::note[What lives elsewhere]
A few values you might expect here are stored separately. **Volume** and **session tokens** live in Nokkvi's internal state database at `~/.local/state/nokkvi/app.redb` so they can update at high frequency without rewriting the config file. **Theme color overrides** live in `~/.config/nokkvi/themes/*.toml` â€” see the [Theming guide](/guides/theming/).
:::

## TOML layout

`config.toml` is divided into a few sections. Keys must be written at the correct nesting level â€” keys placed in the wrong section deserialize as defaults and the change silently no-ops.

| Section | Contents |
| :--- | :--- |
| Root (no header) | `server_url`, `username` |
| `[settings]` | Everything in [General Settings](#general-settings), [Interface Settings](#interface-settings), [Behavior Settings](#behavior-settings), [Playback Settings](#playback-settings), and [Audio Engine](#audio-engine-internal) below |
| `[hotkeys]` | Key rebindings â€” see [Keyboard Shortcuts](/reference/hotkeys/) |
| `[views]` | Per-view sort preferences â€” see [View Sort Preferences](#view-sort-preferences) below |
| `[visualizer]`, `[visualizer.bars]`, `[visualizer.lines]` | Visualizer settings â€” the dotted keys in the tables below (e.g. `visualizer.bars.led_bars`) reflect this nesting |

A minimal file looks like:

```toml
server_url = "https://music.example.com"
username   = "alice"

[settings]
start_view = "Albums"
crossfade_enabled = true

[hotkeys]
slot_list_down = "J"

[views]
albums_sort = "name"

[visualizer]
auto_sensitivity = true

[visualizer.bars]
led_bars = true
```

<Aside type="caution" title="library_page_size: massive">
  Setting `library_page_size` to `"massive"` fetches up to 5,000 items per request. On large libraries this can cause slow initial loads and put significant load on the Navidrome server. Start with `"large"` and only go higher if you need it.
</Aside>

<Aside type="note" title="artwork_resolution: original">
  `"original"` disables server-side resizing and streams the raw embedded artwork from each file. On a large library or a slow connection this can cause noticeable delays when browsing albums. `"ultra"` (2000px) is the practical maximum for most displays.
</Aside>

<Aside type="note" title="Crossfade and normalization">
  In **AGC** mode, tracks that differ much in loudness may show a brief level mismatch at the crossfade boundary because each stream's AGC has not converged yet â€” see [Combining crossfade and normalization](/guides/audio/#combining-crossfade-and-normalization). The two **ReplayGain** modes use static, pre-computed gains instead, so both sides of a crossfade are already at target level â€” no convergence delay, no mismatch.
</Aside>

## Visualizer Settings

Nokkvi features a high-performance GPU visualizer with two main modes: `bars` and `lines`. For a detailed breakdown of color mapping, peak behaviors, and animations, see the [Visualizer Reference](/reference/visualizer/).

### General Visualizer

{/* config-table:start id="visualizer-general" */}
| Key | Default | Description |
| :--- | :--- | :--- |

| `visualizer.noise_reduction` | `0.77` | Smoothing of the raw FFT data (0.0 to 1.0). |
| `visualizer.waves` | `false` | Enable spline smoothing between bars. |
| `visualizer.waves_smoothing` | `5` | Spline control-point density when `waves` is on (2 to 16; higher = smoother curves). |
| `visualizer.monstercat` | `1` | Monstercat-style energy spread to neighbouring bars. Values below `0.7` snap to `0.0` (disabled); effective range is `0.7` to `10.0`. |
| `visualizer.lower_cutoff_freq` | `20` | Bass floor in Hz. Frequencies below this are excluded from the spectrum (minimum `20`). |
| `visualizer.higher_cutoff_freq` | `10000` | Treble ceiling in Hz. Capped at the Nyquist limit (22050). |
| `visualizer.height_percent` | `0.25` | Percentage of window height occupied by the visualizer (0.1 to 0.6). |
| `visualizer.opacity` | `1.0` | Transparency of the visualizer (0.0 to 1.0). |
| `visualizer.bloom` | `true` | Bloom glow post-processing â€” bright bars, peak flashes, and the neon line core bleed a soft additive halo. Applies to all modes. |
| `visualizer.bloom_intensity` | `0.6` | Bloom glow strength (0.0 disables; up to 1.0 for maximum additive glow). |
| `visualizer.beat_reactivity` | `1.0` | How strongly effects pump on the beat and bass drops (0.0 = static/loudness-only, 1.0 = full punch). Scales the bloom surge, neon glow flare, and bar brightness lift together. |
